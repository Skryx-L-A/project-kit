# Copy rules — how posts/ads must be written

These apply to every post and ad this skill produces. The managed project's `brand.md` owns the
product specifics (message order + differentiators); these are the cross-project rules.

## 1. Name the real platforms / targets
State what the product actually supports — e.g. "for Linux and Windows". **Never** use vague
filler like "and beyond", "and more", "everywhere" in place of the concrete list.

## 2. No bare lists
If a post uses a list, put a **short explanatory sentence directly above it**. Never drop a
raw list (first/second/...) with no lead-in.

## 3. Balanced message — lead with what it does
Do not let one angle (e.g. privacy) become the whole post. Default post structure:
1. **What the product does** (the capability) — lead with this.
2. **How/why it is good** — e.g. it runs locally, so it is private and works offline. One line.
3. **Reach / scope** — where it works (e.g. every window, including the terminal).
4. **Differentiators** — 2-4 things that set it apart from alternatives. If listed, add a lead-in.
A recurring important factor (privacy, price, speed) should appear in every post but never be
the entire post.

## 4. Tone
Punchy, show-don't-tell, honest. Prefer concrete demonstration over adjectives. Voice-quote /
"watch this" framing works well. No hype, no invented benchmarks, no emojis unless `brand.md`
explicitly allows.

## 5. Per platform
Adapt length + format to the platform (and each subreddit/forum's rules); never paste identical
text across platforms. Disclose as the maker; FTC #ad on paid.

> The project's `brand.md` should encode the product's exact message order and its concrete
> differentiators so every post is consistent. Generate/maintain that first.
